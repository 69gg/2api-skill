# 项目规范

> 本节是 skill 的强制约定（详见 SKILL.md 第 2 节）。

## 一、命名与目录策略

生成的项目默认命名 **`<平台>2api`**（如 `grok2api`、`promptql2api`、`chathub2api`），除非用户指定别的名字。

目录策略（AI 自动选择，默认直接复制）：

- 若目标目录为空（`.git` 除外），骨架**直接复制到该目录**。
- 若目标目录非空，自动在其下新建 **`<平台>2api`** 子目录并复制到子目录。
- 若目标目录及其 `<平台>2api` 子目录均非空，则中止并提示用户选择新目录，避免覆盖。

## 二、必须用 `copy_skeleton.py` 初始化

**禁止**手写骨架或从其它仓库拷贝当起点。必须调用本 skill 的：

```bash
python /path/to/2api-skill/scripts/copy_skeleton.py --platform <平台> [选项…]
```

### 2.1 功能开关

| 开关 | 默认 | 含义 |
|---|---|---|
| `--with-chat` / `--no-chat` | 开 | `/v1/chat/completions` |
| `--with-responses` / `--no-responses` | 开 | `/v1/responses` |
| `--with-messages` / `--no-messages` | 开 | `/v1/messages`（含 count_tokens） |
| `--with-admin` / `--no-admin` | 开 | `/admin/*` |
| `--with-registrar` / `--no-registrar` | **关** | 注册机目录与相关文档/配置 |
| `--with-email-otp` / `--no-email-otp` | **关** | 邮件验证码收件（隐含 registrar） |
| `--with-captcha` / `--no-captcha` | **关** | 人机验证打码（隐含 registrar） |
| `--init-git` / `--no-init-git` | **关** | 复制后 git init + 标准 ignore + 首提交 |
| `--git-remote <url>` | 空 | 仅 `--init-git` 时设置 origin |
| `--account-dir account\|accounts` | `account` | 账号目录名 |
| `--dest` | `.` | 目标父目录 |

`/v1/models` 与 `/healthz` **始终保留**。

脚本还会：按开关裁剪 FEATURE 锚点块与对应源文件、确保 `.gitignore` 含 `__pycache__/`、写入 `.2api-skill-features.json`。

### 2.2 调用示例

```bash
# 默认当前目录；空则直写，非空则建 grok2api/ 子目录
python scripts/copy_skeleton.py --platform grok

# 只要 Chat + admin，并自动 init git
python scripts/copy_skeleton.py --platform grok --dest ~/projects \
  --no-responses --no-messages --init-git

# 注册机 + 邮件 OTP，无打码
python scripts/copy_skeleton.py --platform grok \
  --with-registrar --with-email-otp --no-captcha
```

第 0 步问清端点与是否 git 后，**一次**把开关传给 copy_skeleton；不要问完 git 却忘记 `--init-git`。

## 三、README 致谢与受众

生成的项目 `README.md` **末尾**必须包含：

```
> 本项目使用 [2api-skill](https://github.com/69gg/2api-skill) 辅助制作。
```

README **只写用户运维向**内容（是什么、特性、快速开始、端点、认证、示例、可选注册机、测试/Docker、目录、License）。**不要**写 IREvent、warmup/guard、真流式状态机、三级解析等内部设计。

## 四、git 忽略（`.gitignore`）

**必须**包含（`copy_skeleton` / `git_init.sh` 会缺则追加；agent 不得删除）：

```
config.toml
.env
account/
accounts/
.venv/
__pycache__/
*.pyc
.pytest_cache/
.ruff_cache/
.mypy_cache/
*.egg-info/
dist/
build/
/tmp/
scripts/probe_out/
```

要点：

- `config.toml` 忽略；`config.toml.example` **不忽略**；
- `account/`（或 `accounts/`）忽略，保留 `*.example`；
- **`__pycache__/` 与 `*.pyc` 必须存在**；
- `git_init.sh` 用法：`bash scripts/git_init.sh [目录] [remote]` 或 `bash scripts/git_init.sh --dir <目录> [--remote <url>]`。

## 五、config.toml 分段

```
[gateway]      # 网关：host/port/api_key
[proxy]        # 代理：url（默认/网关）+ registrar_url（注册机，空则回退 url）；皆空=直连
[upstream]     # 上游行为：timeout/strategy/chat_url/冷却等
[registry]     # 账号目录；registrar 启用时含 target_account_count 等
[admin]        # 管理后台（若 --with-admin）
[email]        # 仅注册机 + 邮件 OTP
[captcha]      # 仅注册机 + 实测有 captcha（可选 proxy_url 覆盖）
```

主程序读 `[gateway]`/`[proxy]`/`[upstream]`/`[registry]`/`[admin]`；`[email]`/`[captcha]` 仅注册机（注册机也读 `[proxy]`）。未启用的段由 copy_skeleton 裁剪，不要手写回「启用态」占位除非实测需要。

**代理回退链**：
- 网关上游：`[proxy].url` → 直连
- 注册机（CLI / 自动补号 / captcha）：CLI `--proxy` → `[proxy].registrar_url` → `[proxy].url` → 直连
- captcha 额外：`[captcha].proxy_url` 显式配置时优先于上述注册机代理

## 六、认证分层

- `/v1/*` 不设 `gateway.api_key` → **无认证**；
- `/admin/*` 不设 `admin.auth_key` → **整个 admin 关闭**（404）；
- 二者独立，详见 `references/auth-and-errors.md`。

## 八、约定式提交（Conventional Commits）

```
feat: 新功能（如 feat: add /v1/responses endpoint）
fix: 修 bug
docs: 文档
test: 测试
refactor: 重构（不改行为）
chore: 杂项
```

`git_init.sh` 首次提交用 `feat: initial <platform>2api scaffold (generated by 2api-skill)`。

## 九、uv 用法

```bash
uv sync                       # 装主依赖
uv sync --extra dev           # 含测试/lint
uv sync --extra registrar     # 含注册机依赖（curl-cffi/playwright）
uv run uvicorn app.main:app --port 8088   # 运行
uv run pytest                 # 测试
uv run ruff check .           # lint
```

## 十、目录布局

```
app/            # FastAPI 服务（写全可运行）
app/upstream/   # 上游适配器（占位，换网站改这里）
registrar/      # 注册机（可选，由 --with-registrar 控制）
tests/          # 单元测试（mock 上游）
config.toml     # 配置（gitignore）
account/        # 账号凭据（gitignore）
.2api-skill-features.json  # copy_skeleton 写入的功能清单（可入库）
```

## 十一、测试与 lint 门槛

- `uv run pytest` 必须全绿（mock 上游，不依赖真实网站）；
- `uv run ruff check .` 必须通过；
- 每完成一个端点立即跑 `scripts/e2e_smoke.py` 冒烟。
