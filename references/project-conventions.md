# 项目规范

> 本节是 skill 的强制约定（详见 SKILL.md 第 2 节）。

## 一、命名

生成的项目默认命名 **`<平台>2api`**（如 `grok2api`、`promptql2api`、`chathub2api`），除非用户指定别的名字。用 `scripts/copy_skeleton.py --platform <平台>` 替换占位。

## 二、README 致谢

生成的项目 `README.md` **末尾**必须包含（markdown 引用块 + 链接）：

```
> 本项目使用 [2api-skill](https://github.com/69gg/2api-skill) 辅助制作。
```

骨架的 `README.md` 已内置此行。

## 三、git 忽略（`.gitignore`）

```
# 凭据与本地配置（勿提交）
config.toml
.env
account/
accounts/

# Python
.venv/
__pycache__/
*.pyc
.pytest_cache/
.ruff_cache/
.mypy_cache/
*.egg-info/
dist/
build/

# 临时
/tmp/
scripts/probe_out/
```

- `config.toml`（含真实凭据/配置）忽略；
- `config.toml.example`（详细注释模板）**不忽略**；
- `account/`（或 `accounts/`，账号凭据目录）忽略，但保留 `*.example`；
- `scripts/git_init.sh` 会自动写入标准 `.gitignore`。

## 四、config.toml 分段

```
[gateway]      # 网关：host/port/api_key
[upstream]     # 上游行为：request_timeout/poll_interval/token_refresh_margin/tool_call_retries/strategy/chat_url
[registry]     # 账号目录：account_dir
[admin]        # 管理后台：auth_key（空=关闭 admin）
[email]        # 仅注册机：cf-temp-email 配置（base_url/admin_auth/custom_auth/domain/poll_timeout）
[captcha]      # 仅注册机：人机验证策略（method/headless/proxy_url/cdp_endpoint/api_provider/api_key）
```

主程序读 `[gateway]/[upstream]/[registry]/[admin]`，忽略 `[email]/[captcha]`（仅注册机用）。

## 五、认证分层

- `/v1/*` 不设 `gateway.api_key` → **无认证**；
- `/admin/*` 不设 `admin.auth_key` → **整个 admin 关闭**（404）；
- 二者独立，详见 `references/auth-and-errors.md`。

## 六、license

生成的项目与配置都用 **MIT**（版权人 `Null <pylindex@qq.com>`）。骨架已含 `LICENSE` 文件。

## 七、约定式提交（Conventional Commits）

```
feat: 新功能（如 feat: add /v1/responses endpoint）
fix: 修 bug
docs: 文档
test: 测试
refactor: 重构（不改行为）
chore: 杂项
```

`scripts/git_init.sh` 首次提交用 `feat: initial <platform>2api scaffold (generated by 2api-skill)`。

## 八、uv 用法

```bash
uv sync                       # 装主依赖
uv sync --extra dev           # 含测试/lint
uv sync --extra registrar     # 含注册机依赖（curl-cffi/playwright）
uv run uvicorn app.main:app --port 8088   # 运行
uv run pytest                 # 测试
uv run ruff check .           # lint
```

## 九、目录布局

```
app/            # FastAPI 服务（写全可运行）
app/upstream/   # 上游适配器（占位，换网站改这里）
registrar/      # 注册机（可选）
tests/          # 单元测试（mock 上游）
config.toml     # 配置（gitignore）
account/        # 账号凭据（gitignore）
```

## 十、测试与 lint 门槛

- `uv run pytest` 必须全绿（mock 上游，不依赖真实网站）；
- `uv run ruff check .` 必须通过；
- 每完成一个端点立即跑 `scripts/e2e_smoke.py` 冒烟。
